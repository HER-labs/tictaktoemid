<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → wa.me (WhatsApp) | External navigation to WhatsApp with pre-filled message |
| Browser → instagram.com | External link opens Instagram profile in new tab |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Instagram footer link (`target="_blank"`) | mitigate | `rel="noopener noreferrer"` on the `<a>` element (prevents reverse tabnapping) |
| T-02-02 | Tampering | WhatsApp pre-fill URL text parameter | accept | Pre-fill is a static encoded string with no user input; no injection vector |
| T-02-03 | Information Disclosure | WhatsApp phone number in HTML source | accept | Phone number is intentionally public as business contact; placeholder `65XXXXXXXX` used until confirmed |
| T-02-04 | Denial of Service | Skip link focus trap | accept | Skip link uses standard focus pattern; no custom trap logic that could break |
</threat_model>
